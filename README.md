# Multi-Domain Support Triage Agent

## 🚀 Project Overview
This project is an automated support ticket triage agent built for the **HackerRank Orchestrate Challenge**. It processes support tickets for three ecosystems: **HackerRank, Claude, and Visa**, ensuring that issues are categorized correctly and high-risk cases are escalated to humans.

## 🛠 Features
- **Automatic Domain Inference:** Identifies the company (HackerRank, Claude, or Visa) even if the data is missing.
- **Risk Assessment:** Automatically escalates tickets involving billing, fraud, or account security.
- **Safety First:** Only uses provided support documentation to generate responses (no hallucinations).
- **Batch Processing:** Handles large CSV files and outputs structured results.

## 📂 File Structure
- `triage_agent.py`: The main Python script containing the triage logic.
- `support_issues.csv`: The input file containing support tickets.
- `output_results.csv`: The final triaged output generated by the agent.
